Creating DB2 warehouse instance in C4PD

Once Db2 warehouse operators related resources are installed and configured on CP4D,
You can create an instance through CP4D web console.

Check prerequite resources#

There are multiple types of resources.
Checking each, the followings will show up.
The majority of those are operators that helps to configure DB2/Db2wh/DMC instances and catalog/subscription related resources.

The tools are available for download from cpd-cli github page.

Login CP4D web console using the credentials.

From the hone page, click View all in the My instance box.
Or click Databases in the left pane.

The page shows the current database list.

Click Create a database at right top.

Create DB2 instance#

From the availalbe services, select Db2 warehouse.


Select options following the pages and click Next.
The followings are few screen examples.
Pages detail may change depending on versions.








Review the selections and click create.

Monitor the progress or coffee time until the completion.

You may want to check until db2wh pods are Running status.

Failure with insufficient memory#

In case memory is not enough, it refuses to schedule. 

[root@api.jscp4d.cp.fyre.ibm.com v4.7]# oc describe pod c-db2oltp-1697933778158541-db2u-0...Events:  Type     Reason            Age                     From               Message  ----     ------            ----                    ----               -------  Warning  FailedScheduling  8m4s (x247 over 3h12m)  default-scheduler  0/6 nodes are available: 3 Insufficient memory, 3 node(s) had untolerated taint {node-role.kubernetes.io/master: }. preemption: 0/6 nodes are available: 3 No preemption victims found for incoming pod, 3 Preemption is not helpful for scheduling.

In my case, it failed on 3 worker nodes that has 32 GB memory each.
